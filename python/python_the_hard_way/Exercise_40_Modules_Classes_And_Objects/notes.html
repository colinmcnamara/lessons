Classes are like modules

Modules are a specialized dictionary that can store python code so you can get to it with the . operator.

Classes are similar

A class is a way to take a grouping of functions AND DATA and place them in a container that you can access with the . operator.

Python class checklist

Example code to reference


class MyStuff(object):

    def __init__(self):
        self.tangerine = "And now a thousand years between"

    def apple(self):
        print "I AM CLASSY APPLES!"
thing = MyStuff()
thing.apple()
print thing.tangerine

  1. Python looks for MyStuff() and sees it’s a class

  2. Python creates an empty object with all the functions that you have specified in the class using def

  3. Python then looks for the "Magic" +init function, and calls that function to intialize your empty object

  4. In the MyStuff() function init we then get this extra variable self which is that empty object python made for me. and we can set variables on it just like you would a module,dict or other object.

  5. In this case we set self.tangerine to a song lyric and then itialized it.

  6. Then we took the object, assigned it to thing for us to call, and print

More important thoughts on classes

  1. Classes are like bluerprint or definitions for creating new mini-modules

  2. Instantiation is how you make on and import it at the same time

  3. The resulting created mini-module is an object which you thne assign variables to work with it.
